Paradigma orientado a servicios

Desarrollo de aplicaciones orientadas a servicios

Este enfoque construye soluciones a partir de servicios independientes, reutilizables y accesibles desde la red, sin importar la plataforma o lenguaje.

Servicios en la nube

  • IaaS: provee infraestructura como servidores, almacenamiento y redes (por ejemplo, maquinas virtuales).
  • PaaS: entrega plataformas completas para desarrollar, probar y desplegar aplicaciones.
  • SaaS: ofrece software listo para usarse a traves de Internet, como correo o ERPs.

Caracteristicas claves

  • Desacoplamiento, reutilizacion e interoperabilidad entre servicios.
  • Escalabilidad y acceso remoto mediante protocolos como HTTP.

Aplicaciones hibridas (Mashups)

  • Integran APIs externas para combinar datos en una sola interfaz.
  • Mejoran la experiencia del usuario unificando mapas, clima, trafico u otras fuentes.

Arquitectura SOA

Definicion y principios

SOA organiza las aplicaciones como un conjunto de servicios autonomos con interfaces bien definidas que se comunican por protocolos estandar.

  • Cada servicio atiende una tarea especifica y puede reutilizarse en distintos sistemas.
  • Principios: bajo acoplamiento, alta cohesion, reutilizacion, autonomia, descubribilidad e interoperabilidad.
3

Estándares relacionados

Protocolos y contratos

Dominios esenciales para cualquier implementacion orientada a servicios.

  • XML: lenguaje para estructurar y transportar datos.
  • SOAP: protocolo basado en XML para mensajeria sobre HTTP/HTTPS.
  • WSDL: describe operaciones, mensajes y formatos de un servicio.
  • UDDI: registro para publicar y descubrir servicios disponibles.
  • REST: estilo arquitectonico que usa metodos HTTP como GET, POST, PUT y DELETE.
